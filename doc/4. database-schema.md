# Database Schema – FreelancerOS

This document defines the data structure for the FreelancerOS MVP using Convex.

All primary entities include a `userId` field to support multi-tenant architecture.

---

## 1. Users

Handled by Convex Auth.

User data reference:

- \_id
- email
- name (optional)

---

## 2. Clients

Stores freelancer client information.

Fields:

- \_id
- userId (reference to user)
- name (string)
- email (string)
- phone (optional string)
- company (optional string)
- notes (optional string)
- createdAt (number)

---

## 3. Projects

Each project belongs to a client.

Fields:

- \_id
- userId
- clientId (reference to Clients)
- title (string)
- description (optional string)
- status (Active | Completed | On Hold)
- startDate (number)
- deadline (optional number)
- createdAt (number)

Relationship:
Client → Projects (one-to-many)

---

## 4. Tasks

Tasks belong to a project.

Fields:

- \_id
- userId
- projectId (reference to Projects)
- title (string)
- description (optional string)
- priority (Low | Medium | High)
- status (Pending | Completed)
- dueDate (optional number)
- createdAt (number)

Relationship:
Project → Tasks (one-to-many)

---

## 5. Invoices

Invoices track payments for projects.

Fields:

- \_id
- userId
- projectId (reference to Projects)
- amount (number)
- status (Paid | Unpaid)
- issueDate (number)
- paidDate (optional number)
- createdAt (number)

Relationship:
Project → Invoices (one-to-many)

---

## 6. Entity Relationships Overview

User
├── Clients
│ └── Projects
│ ├── Tasks
│ └── Invoices
